Production Pipeline: 8 этапов

Формат: Воркшоп проходит все 8 этапов. Ключевые (3, 4, 7, 8) — глубоко, остальные — обзорно. Ведущий строит параллельно с участниками. Все материалы — в репо.

00:10-00:15

2Бизнес-анализ

00:15-00:25

3ТЗ на сайт

Обзорно: структура, секции, контент-план, стек. Claude генерирует SITE_SPEC.md.

Материал: pipeline/03-site-spec.md

00:25-01:25

4Реализация

Глубоко (60 мин): Hero + секции (prompts/01-site.md) → premium-эффекты (prompts/02-effects.md). Checkpoints по ходу.

Материал: pipeline/04-build.md + types/{тип}/prompts/01-site.md, 02-effects.md

CP1 (00:40) — Hero готов

01:25-01:30

01:30-01:35

Обзорно: формы, responsive, кроссбраузерность. QA-агент в отдельной сессии.

Глубоко: SEO-промпт (03-seo.md). Security чек-лист. Privacy policy. Отдельная сессия для security!

01:45-01:55

01:55-02:00

Финал

2-3 волонтёра показывают результат. Ссылка на репо. Q&A.

Заметки ведущего

  • Этапы 1-2 и 5-6 — обзорные, показываем подход и материалы. Участники делают глубже дома.
  • Этапы 3-4 и 7-8 — основное время. Здесь участники строят руками.
  • Кто-то отстаёт — дай конкретный промпт, не объясняй. «Скопируй вот это».
  • Claude зависает — подожди 30 сек, потом новая сессия с тем же промптом.
  • Эффекты не работают — проверь консоль. 90% проблем — GSAP/ScrollTrigger/Lenis не подключён.
  • Время гибкое — быстро идёт? Углубляемся в контент. Медленно? Пропускаем QA.
  • Pipeline-доски — в папке active/boards/ по доске на каждый этап.

Pipeline: все этапы с промптами

1Инфраструктура

Файл: pipeline/01-infrastructure.md

Склонируй репозиторий с материалами воркшопа: https://github.com/dkoraitest/premium-site-workshop.git Установи skills для качественной работы: 1. greensock/gsap-skills — правильный GSAP, ScrollTrigger, performance 2. taste-skill от Leonxlnx — premium UI quality 3. grill-me от mattpocock — stress-test планов и решений Создай папку для моего проекта, инициализируй git, добавь .env* в .gitignore.

2Бизнес-анализ

Skills: brainstorming → grill-me | Файл: pipeline/02-business-analysis.md

Используй skill /brainstorming для генерации идей, затем /grill-me для прожарки результата. Проведи бизнес-анализ для создания премиум-сайта. Результат оформи как BUSINESS_ANALYSIS.md. Бизнес: [НАЗВАНИЕ], сфера: [СФЕРА], продукт: [ЧТО ПРОДАЁТЕ], ценовой сегмент: [ПРЕМИУМ/СРЕДНИЙ]. Определи: 1. Целевую аудиторию (кто, какие боли, где ищет решение) 2. Ценностное предложение (через какую боль транслируем) 3. Анализ 2-3 конкурентов (что у них хорошо/плохо) 4. Рекомендацию типа сайта: video-hero / saas-aurora / agency-kinetic / product-scroll 5. Бренд-атрибуты: цвета, настроение, тон коммуникации

3ТЗ на сайт

Skill: grill-me (прожарка ТЗ) | Файл: pipeline/03-site-spec.md

На основе бизнес-анализа из BUSINESS_ANALYSIS.md сгенерируй полное ТЗ на сайт — SITE_SPEC.md: 1. Структура: какие страницы, какие секции на каждой 2. Контент-план: что нужно для каждой секции (текст, фото, видео) 3. Функциональные требования (формы, кабинеты, интеграции) 4. Выбор стека с обоснованием 5. Дизайн-направление: 2-3 сайта-референса, mood, палитра 6. SEO-стратегия: ключевые запросы, структура URL После генерации используй skill /grill-me чтобы прожарить структуру ТЗ и найти пробелы.

Используй taste-skill для premium качества UI и gsap-skills для анимаций. Создай лендинг для [БИЗНЕС/БРЕНД]. Стиль: кинематографичный, luxury. Шаг 02: Premium-эффекты

Используй gsap-skills для правильных анимаций и orchestrating-gsap-lenis для совместимости ScrollTrigger + Lenis. Добавь premium-эффекты: 1. Scroll-scrub видео: при скролле видео проигрывается вперёд/назад (GSAP ScrollTrigger). Загрузи видео как blob через XHR. Smooth seeking: targetTime + requestAnimationFrame + lerp (0.06). 2. Film grain overlay: полноэкранный div с CSS noise-текстурой, opacity 0.03-0.05, pointer-events: none. 3. Fade-in анимации: все секции с translateY(30px) + opacity через IntersectionObserver. 4. Hero текст fade-out: opacity 1→0, translateY вверх через ScrollTrigger.

5Контент

Файл: pipeline/05-content.md

Используй установленные skills (taste-skill для качества UI, gsap-skills для анимаций). Замени все placeholder-тексты на реальный контент: 1. Сгенерируй тексты на основе бизнес-анализа из BUSINESS_ANALYSIS.md: заголовки, описания секций, CTA-кнопки. Тон — [ПРОФЕССИОНАЛЬНЫЙ / ДРУЖЕЛЮБНЫЙ / LUXURY]. 2. Hero-заголовок: транслируй главную ценность через боль ЦА. 3. Секция features: по 1-2 предложения на каждую фичу, конкретные выгоды. 4. CTA: конкретное действие, не «Узнать больше», а «Получить демо» / «Рассчитать стоимость». 5. Оптимизируй изображения: конвертируй в WebP (q80), добавь loading="lazy" для below-fold.

6Проверка (QA)

Файл: pipeline/06-qa-check.md | Запускай в отдельной сессии!

Запусти в ОТДЕЛЬНОЙ сессии Claude Code (QA-агент). Проведи QA-проверку сайта. Проверь каждый пункт, отчитайся PASS/FAIL/WARN: 1. Responsive: 375px (iPhone SE), 768px (iPad), 1440px (Desktop) 2. Нет горизонтального скролла на мобильных 3. Все ссылки работают (нет 404) 4. Все изображения загружаются 5. Формы: отправка работает, валидация, success/error сообщения 6. Анимации плавные (нет jank), 60fps 7. Навигация работает (якорные ссылки) 8. Кроссбраузерность: Chrome, Safari, Firefox 9. Нет Lorem ipsum или placeholder-текстов 10. Footer с актуальными контактами

7Security + SEO + Legal

Файл: pipeline/07-security-seo.md | Security — ОТДЕЛЬНАЯ сессия!

SEO

Добавь SEO-обвязку: 1. Meta: title (до 60 символов), description (до 160), canonical URL 2. Open Graph: og:title, og:description, og:image (1200x630, абсолютный URL), og:url, og:type, og:locale ru_RU 3. Twitter Card: summary_large_image 4. JSON-LD: Organization + WebSite + WebPage (связь через @id) 5. sitemap.xml, robots.txt (разреши GPTBot, ClaudeBot, PerplexityBot), llms.txt

Security (новая сессия!)

Запусти в ОТДЕЛЬНОЙ сессии Claude Code (Security-агент). Агент, который писал код, предвзят к своим ошибкам. Проведи security-аудит по чек-листу checklists/security.md. Проверь каждый пункт, отчитайся PASS/FAIL: - .env в .gitignore, нет hardcoded ключей - XSS: все данные из форм экранируются - Кнопка disable при отправке, серверная валидация - HTTPS, HTTP→HTTPS redirect - Внешние скрипты с конкретными версиями (не latest)

Legal

Добавь Privacy Policy (ФЗ-152 для РФ) и Cookie Consent баннер. Баннер фиксирован внизу, кнопка принятия + ссылка на policy, запоминает выбор в localStorage. Не загружай аналитику до принятия.

8Деплой + Аналитика

Файл: pipeline/08-deploy-analytics.md

Формы → Telegram

Подготовка: @BotFather → /newbot → токен. Добавь бота в группу. Получи chat_id: api.telegram.org/bot{TOKEN}/getUpdates. Создай .env.
Добавь форму обратной связи с отправкой в Telegram: 1. HTML-форма: имя, телефон, email (обязательные), комментарий (опционально). 2. Serverless function api/submit.js: валидация, escape HTML, POST в Telegram Bot API. 3. Клиентский JS: fetch POST, disabled кнопка, success/error сообщения. 4. Токены из process.env (TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID).

Деплой

Подготовь проект к деплою на Cloudflare Pages: 1. Все пути относительные, .env в .gitignore 2. Serverless → functions/api/submit.js (формат Cloudflare Workers) 3. git init, git add, git commit 4. gh repo create my-site --public --push
dash.cloudflare.com → Pages → Create → Connect GitHub → Deploy. Добавь env vars в Settings.

Яндекс.Метрика

Добавь счётчик Яндекс.Метрики. ID: [METRIKA_ID]. Код перед </head>. Включи: webvisor: true, clickmap: true, accurateTrackBounce: true. Добавь noscript-пиксель в body. Настрой цели: ym(ID, 'reachGoal', 'cta_click') на CTA, ym(ID, 'reachGoal', 'form_submit') на форму.

Google Analytics 4

Добавь GA4 через gtag.js. ID: [GA4_ID]. Код в head. Настрой события: gtag('event', 'cta_click', {section: 'hero'}), gtag('event', 'generate_lead', {form_name: 'contact'}). Скролл 50%/90% — через enhanced measurement.
