# Этап 1: Бизнес-анализ

Прежде чем писать код — разберись в бизнесе. Этот этап определяет всё остальное: структуру сайта, тон текстов, визуальный стиль, выбор типа.

---

## Промпт для Claude Code

> Проведи бизнес-анализ для создания премиум-сайта. Результат оформи как структурированный документ `BUSINESS_ANALYSIS.md`.
>
> **Бизнес:**
> - Название: [НАЗВАНИЕ КОМПАНИИ / БРЕНДА]
> - Сфера: [СФЕРА ДЕЯТЕЛЬНОСТИ — например: IT-консалтинг, производство мебели, онлайн-образование]
> - Основной продукт/услуга: [ЧТО ИМЕННО ПРОДАЁТЕ / ДЕЛАЕТЕ]
> - География: [ГОРОД / РЕГИОН / СТРАНА / МЕЖДУНАРОДНЫЙ]
> - Ценовой сегмент: [ЭКОНОМ / СРЕДНИЙ / ПРЕМИУМ / LUXURY]
> - Текущий сайт (если есть): [URL]
>
> **Целевая аудитория:**
> - Кто принимает решение о покупке: [РОЛЬ — CEO, маркетолог, частное лицо, закупщик]
> - Демография: [ВОЗРАСТ, ПОЛ, ДОХОД, ОБРАЗОВАНИЕ — если известно]
> - Главные потребности аудитории: [ЧТО ИМ НУЖНО — экономия времени, статус, надёжность, инновации]
> - Главные боли: [ЧТО МЕШАЕТ / ФРУСТРИРУЕТ — дорого, долго, непонятно, нет доверия]
> - Где они ищут решение: [ПОИСК GOOGLE/YANDEX, РЕКОМЕНДАЦИИ, СОЦСЕТИ, ТЕНДЕРЫ]
>
> **Конкуренты (2-3 примера для анализа):**
> - [URL КОНКУРЕНТА 1] — что у них хорошо / что плохо
> - [URL КОНКУРЕНТА 2] — что у них хорошо / что плохо
> - [URL КОНКУРЕНТА 3] — что у них хорошо / что плохо
>
> **Ценностное предложение:**
> - В чём уникальность: [ЧЕМ ОТЛИЧАЕТЕСЬ ОТ КОНКУРЕНТОВ]
> - Главная ценность для клиента: [ЧТО ПОЛУЧАЕТ КЛИЕНТ — не фича, а результат]
> - Почему должны выбрать именно вас: [1-2 ПРЕДЛОЖЕНИЯ]
>
> **Бренд:**
> - Фирменные цвета (если есть): [HEX-КОДЫ ИЛИ ОПИСАНИЕ — тёмно-синий, золотой]
> - Настроение бренда: [СЕРЬЁЗНЫЙ / ДРУЖЕЛЮБНЫЙ / ТЕХНОЛОГИЧНЫЙ / LUXURY / МИНИМАЛИСТИЧНЫЙ]
> - Тон коммуникации: [ФОРМАЛЬНЫЙ / ПОЛУФОРМАЛЬНЫЙ / НЕФОРМАЛЬНЫЙ]
> - Референсы по стилю (если есть): [URL САЙТОВ, КОТОРЫЕ НРАВЯТСЯ]
>
> На основе этих данных:
> 1. Сформулируй positioning statement (1 предложение)
> 2. Определи ключевые сообщения для сайта (3-5 штук)
> 3. Опиши Jobs To Be Done (JTBD) для посетителя сайта: зачем он пришёл, что хочет понять, какое действие должен совершить
> 4. Проанализируй конкурентов: что взять лучшее, чем отстроиться
> 5. Рекомендуй тип сайта из доступных:
>    - **video-hero** — кинематографичный, luxury, технологии, недвижимость
>    - **saas-aurora** — gradient + glass, SaaS, AI, fintech
>    - **agency-kinetic** — text reveal + cursor, агентства, портфолио
>    - **product-scroll** — horizontal pin + parallax, продукты, e-commerce
> 6. Обоснуй выбор типа: почему именно этот подходит для данного бизнеса и аудитории
> 7. Предложи цветовую палитру (primary, secondary, accent, background, text) с HEX-кодами
> 8. Предложи шрифтовую пару (заголовки + текст) из Google Fonts

## Что заполнить

| Placeholder | Что писать | Пример |
|---|---|---|
| `[НАЗВАНИЕ КОМПАНИИ / БРЕНДА]` | Юрлицо или торговое название | AI Surfers |
| `[СФЕРА ДЕЯТЕЛЬНОСТИ]` | Индустрия | Корпоративное обучение AI |
| `[ЧТО ИМЕННО ПРОДАЁТЕ]` | Конкретный продукт/услуга | Воркшопы по AI для бизнеса |
| `[ГОРОД / РЕГИОН / СТРАНА]` | Где работаете | Москва, РФ + международные проекты |
| `[ЭКОНОМ / СРЕДНИЙ / ПРЕМИУМ / LUXURY]` | Ценовой сегмент | Премиум |
| `[РОЛЬ]` | Кто ваш buyer persona | HRD, CTO, CEO малого и среднего бизнеса |
| `[ВОЗРАСТ, ПОЛ, ДОХОД]` | Если знаете — укажите | 30-50, M/F, высокий |
| `[ПОТРЕБНОСТИ]` | Что клиенту нужно | Быстро внедрить AI в процессы |
| `[БОЛИ]` | Что фрустрирует | Сотрудники не умеют использовать AI |
| `[URL КОНКУРЕНТОВ]` | Сайты конкурентов | site1.ru, site2.com |
| `[УНИКАЛЬНОСТЬ]` | Чем отличаетесь | Практический подход, не теория |
| `[ГЛАВНАЯ ЦЕННОСТЬ]` | Результат для клиента | Команда начинает использовать AI через 1 день |
| `[ФИРМЕННЫЕ ЦВЕТА]` | Если есть — HEX-коды | #0A1628, #14B8A6, #FFF5E1 |
| `[НАСТРОЕНИЕ]` | Атмосфера бренда | Технологичный, но доступный |
| `[ТОН]` | Как общаетесь | Полуформальный, экспертный |
| `[РЕФЕРЕНСЫ]` | Сайты, которые нравятся | linear.app, vercel.com |

- Рекомендацией типа сайта с обоснованием

- Шрифтовой парой

Этот документ — вход для следующего этапа (02-site-spec).

## Stress-test через grill-me

После генерации бизнес-анализа — прожарь его через скилл grill-me:

```bash
npx skills add mattpocock/skills --skill "grill-me"
```

Затем попроси Claude: `/grill-me` — и он будет задавать вопросы по каждой ветке: «Точно ли эта ЦА? А если конкурент X сделает Y?». Помогает найти пробелы до начала работы.

---

## Опциональное расширение

Если бизнес сложный или конкурентная среда плотная, добавь к промпту:

> Дополнительно:
> - Проведи SWOT-анализ (Strengths, Weaknesses, Opportunities, Threats) для онлайн-присутствия
> - Определи 3 ключевых возражения аудитории и как сайт должен их снимать
> - Предложи CTA-стратегию: какое главное действие (заявка, звонок, демо, регистрация) и как к нему вести
> - Определи метрики успеха сайта: что считать конверсией, какой показатель отслеживать
