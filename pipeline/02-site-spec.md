# Этап 2: Спецификация сайта (ТЗ)

На основе бизнес-анализа (этап 01) генерируем полное техническое задание: структура, контент, функционал, дизайн.

---

## Промпт для Claude Code

> На основе бизнес-анализа из файла `BUSINESS_ANALYSIS.md` создай полную спецификацию сайта в файле `SITE_SPEC.md`.
>
> **Входные данные:**
> - Бизнес-анализ: прочитай `BUSINESS_ANALYSIS.md`
> - Тип сайта: [ТИП ИЗ АНАЛИЗА — video-hero / saas-aurora / agency-kinetic / product-scroll]
> - Количество страниц: [ОДНОСТРАНИЧНИК / МНОГОСТРАНИЧНЫЙ — сколько страниц]
> - Язык контента: [РУССКИЙ / АНГЛИЙСКИЙ / МУЛЬТИЯЗЫЧНЫЙ]
>
> Спецификация должна включать:
>
> ### 1. Структура сайта
> Карта страниц и секций. Для каждой страницы:
> - Название страницы и URL
> - Список секций сверху вниз
> - Назначение каждой секции (что посетитель должен понять / почувствовать / сделать)
> - Приоритет секции (must-have / nice-to-have)
>
> Для лендинга стандартная структура:
> 1. Hero (захват внимания, positioning statement, CTA)
> 2. Pain/Problem (боль аудитории, зачем им это нужно)
> 3. Solution/About (что вы предлагаете, как решаете боль)
> 4. Features/Services (3-6 ключевых преимуществ / услуг)
> 5. Social Proof (кейсы, клиенты, отзывы, цифры)
> 6. How It Works / Process (3-5 шагов — как начать)
> 7. Pricing / Packages (если применимо)
> 8. Team (если добавляет доверия)
> 9. FAQ (снятие типичных возражений)
> 10. CTA + Contact (форма, контакты, призыв к действию)
> 11. Footer (навигация, юрлицо, соцсети, политика)
>
> ### 2. Контент-план
> Для каждой секции:
> - Заголовок (H1/H2): формулировка или направление
> - Подзаголовок: 1-2 предложения
> - Основной текст: объём (сколько абзацев, пунктов)
> - Визуал: что нужно (фото, иллюстрация, видео, иконка, скриншот, 3D)
> - CTA: текст кнопки, куда ведёт
>
> ### 3. Функциональные требования
> - Формы: какие поля, куда отправка, валидация
> - Интеграции: Telegram-бот, CRM, аналитика, платёжная система
> - Интерактив: калькулятор, квиз, фильтры, поиск, личный кабинет
> - Модальные окна: когда открываются, что содержат
> - Навигация: якорные ссылки, burger menu, sticky header
>
> ### 4. Технический стек
> Обоснуй выбор:
> - Фреймворк: Vanilla HTML/CSS/JS, React, Next.js, Astro — и почему
> - Анимации: GSAP + Lenis (стандарт воркшопа)
> - Стили: чистый CSS, Tailwind — и почему
> - Хостинг: Cloudflare Pages (РФ), Netlify, Vercel — и почему
> - Формы: serverless → Telegram
>
> ### 5. Дизайн-направление
> - Цветовая палитра (из бизнес-анализа): primary, secondary, accent, background, text — HEX
> - Шрифтовая пара: заголовки + текст (Google Fonts)
> - Стиль: минималистичный / насыщенный / luxury / tech
> - Референсы: 2-3 сайта с пояснением что именно взять (layout, анимации, цвет, типографика)
> - Визуальные механики из MECHANICS.md: какие применить и где
>
> ### 6. SEO-стратегия
> - Целевые ключевые запросы (3-5 основных)
> - Title и description для каждой страницы
> - Schema-тип: Organization, LocalBusiness, SoftwareApplication, Event — что подходит
> - Контент-стратегия: будет ли блог, кейсы, страницы услуг

Файл `SITE_SPEC.md` — полное ТЗ, по которому можно строить сайт без дополнительных вопросов.

Содержит:
- Карту страниц и секций с описанием назначения каждой
- Контент-план: что писать в каждой секции, какой визуал нужен

## Опциональные дополнения

### Если нужен DESIGN.md

> На основе спецификации из `SITE_SPEC.md` создай файл `DESIGN.md` в формате awesome-design-md. Включи: цветовую палитру (CSS-переменные), типографику (шрифты, размеры, line-height для каждого уровня), spacing-систему (4px grid), border-radius, shadows, breakpoints, компоненты (кнопки, карточки, формы, навигация). Всё в виде готовых CSS-переменных и классов.

### Если многостраничный сайт

Добавь к промпту:

> Для многостраничного сайта дополнительно:
> - Определи навигационную иерархию (header menu, footer links)
> - Для каждой внутренней страницы: цель, ключевой запрос, контент, CTA
> - Стратегия перелинковки: как страницы ссылаются друг на друга
> - View Transitions: какие элементы анимировать между страницами
