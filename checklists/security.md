# Чек-лист безопасности

Минимальный security-pass перед публикацией сайта. Основан на опыте создания AI Builder (aibuildermsk.ru).

## ENV и секреты

- [ ] `.env` в `.gitignore` — токены не попадают в git
- [ ] В репо нет hardcoded API ключей, токенов, паролей
- [ ] На хостинге env-переменные заданы через dashboard (не в коде)
- [ ] Нет console.log с чувствительными данными

## XSS и инъекции

- [ ] Все данные из формы экранируются перед вставкой в HTML (`<` → `&lt;`, `>` → `&gt;`, `&` → `&amp;`)
- [ ] Нет `innerHTML` с пользовательским вводом
- [ ] Serverless function валидирует входные данные (обязательные поля, типы)
- [ ] Content-Type проверяется на сервере (ожидаем application/json)

## Формы

- [ ] Кнопка отправки disable на время запроса (защита от double-submit)
- [ ] Серверная валидация (не только клиентская)
- [ ] Ошибки не раскрывают внутреннюю информацию (stack traces, пути, токены)
- [ ] Rate limiting: хотя бы клиентский (disable кнопки + таймер)

## Заголовки и HTTPS

- [ ] HTTPS работает (автоматически на Netlify/Vercel/Cloudflare)
- [ ] HTTP → HTTPS redirect настроен
- [ ] X-Content-Type-Options: nosniff (если есть доступ к headers)
- [ ] X-Frame-Options: DENY (если сайт не должен встраиваться в iframe)

## Зависимости

- [ ] Нет подключений скриптов с подозрительных CDN
- [ ] Внешние скрипты (GSAP, Lenis) подключаются с конкретными версиями, не latest
- [ ] Если используется npm — `npm audit` не показывает critical vulnerabilities

## Видео и медиа

- [ ] Видео не содержит метаданных с геолокацией или личными данными
- [ ] Изображения очищены от EXIF-данных (если есть личные фото)
